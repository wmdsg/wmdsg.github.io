前言

堆排序在面试中尤为常见，虽然许多语言中有现成的优先级队列或堆的容器使用，但面试官更倾向于自己实现堆，并进行堆排序，故下文开始详细阐述堆排序这个知识点。

什么是堆

在介绍什么是堆之前先来介绍一下优先级队列。我们在某些情况下希望数据集合是有序的，但并不要求全部有序或者在操作开始前就完全有序，在许多应用中需先收集一部分数据，从中挑选具有最大关键码(最小)的记录开始处理，对于这类应用，我们期望的数据结构能支持插入操作，并能方便取出最大(最小)关键码记录。这样的数据结构称为优先级队列。要构造高效率的优先级队列，堆是最高效的一种方式。

堆通常是可以被看做是完全二叉树的数据结构。

其满足下列性质：

  • 堆中某个结点的值总是不大于或不小于其父节点的值
  • 堆是完全二叉树

根节点大的堆叫最大堆，根节点小的堆叫最小堆。如下给出最大堆示例

堆的根节点存放的是最大或最小元素，但其他节点的排列顺序是未知的。在一个最大堆中，最大的元素总是在index0的位置，但最小的元素不一定是最后一个元素，唯一能保证的是最小元素是一个叶节点。这也正是优先级队列的初衷，并不需要全部有序，只要你给我最大或最小元素即可。

堆属性

介绍了什么是堆，我们就要思考该如何建立一个堆呢？建立一个堆首先得搞清楚其各个节点的位置关系。

  • 如果i=0，则表示是根节点，否则，父节点：parent(i)=(i-1)/2
  • 如果2i+1>n-1则结点i无左子节点，否则左子节点位置为2i+1;
  • 如果2i+2>n-1则无右子女，否则结点i的右子女为2i+2；

堆的建立

基于堆是完全二叉树，故可用数组实现堆。

public class MinHeap<E extends Comparable>{
    private Array<E> data;
    public MinHeap(int capacity){
        data=new Array<>(capacity);
    }
    
    public int size(){
        return data.getSize();
    }
    
    public boolean isEmpty(){
        return data.isEmpty();
    }
    
    public int parent(int index){
        return (index-1)/2;
    }
    
    public int leftChild(int index){
        return index*2+1;
    }
    
    public int rightChild(int index){
        return index*2+2;
    }
}

堆插入

为维持完全二叉树属性，新插入的元素一定是在最后一个结点，同时为了满足堆的值的约束属性，新插入的元素要与父节点比较，比父节点小就把父节点拉下来代替当前结点位置，自己则不断向上寻找，直到比父节点大为止。

//添加一个辅助函数用来交换两个位置的值。
public void swap(int i,int j){
	if(i<0||i>=size||j<0||j>=size){
        throw new IllegalArgumentException("index is illegal");
    }	
    E temp=data[i];
    data[i]=data[j];
    data[j]=temp;
}
//堆添加元素
public void insert(E e){
	if(currentSize==maxSize){
        throw new OutOfSizeException("heap is full");
    }
    data[currentSize]=e;
    siftUp(currentSize);//此时currentSize并没有加一,传入的是索引
    currentSize++;
}

private void siftUp(int i){
    //比较插入值和父节点的大小关系。
    while(i>0){
        if(data.get(i).compareTo(data.get(parent(i)))<0){
            data.swap(i,parent(i));
            i=parent(i);//i来到新位置后继续向上比较
        }else{
            break;
        }
    }
}

堆删除

堆删除即将最后一个元素填充到堆顶，为了维持堆的特性，不断下沉这个元素。

假设要从节点 1 ，也可以称为取出节点 1 ，为了维持完全二叉树的特性 ，我们将最后一个元素 6 去替代这个 1 ；然后比较 1 和其子树的大小关系，如果比左右子树大（如果存在的话），就要从左右子树中找一个较小的值替换它，而它能自己就要跑到对应子树的位置，再次循环这种操作，直到没有子树比它小。

  • 找到要删除的结点的位置
  • 数组中最后一个元素替代这个元素位置。
  • 下滑
  • 删除最后一个元素。
public E findMin(){
    return data.get(0);
}

public E removeMin(){
    //可以先判断堆是否为空，这里就不实现了
    E ret=findMin();
    data[0]=data[data.getSize()-1];
    data.removeLast();
    siftDown(0);//也可以把currentSize-1传进去，传的是索引，表示在[0,currentSize-1]内调整堆
    return ret;
}

public void siftDown(int start){
    while(leftChild(start)<data.getSize()){
        int j=leftChild(start);
        if(j+1<data.getSize()&&data[j]>data[j+1]) j++;
        if(data[start]<=data[j]) break;
        data.swap(start,j);
        start=j;
    }
}

堆的建立

让一个已知数组形成堆的方式是通过一个记录数组复制原数组，并对这个记录数组不断siftDown调整，形成堆。

public MinHeap(E[] arr,int n){
    data=new Array<>(n);
	for(int i=0;i<n;i++) data[i]=arr[i];
    int currentPos=n-1;
    while(currentPos>=0){
        siftDown(0);//end为数组长度-1
        currentPos--;//局部自上而下下滑调整
    }
}

时间复杂度

n个结点的完全二叉树深度为$k=log_2^{n+1}$取上界，堆的调整算法siftDown，while循环次数最大为树的深度减1，所以堆的删除算法时间复杂度为$O(log_2^n)$,

插入新结点siftUp时，while的循环次数不超过数的深度减1，故也为$O(log_2^n)$，

堆的建立操作siftDown执行了n/2次，故时间复杂度为$O(nlog_2^n)$.

堆排序

过程如下：

  1. 先建立初始堆
  2. 重复删除堆顶元素
public class heapsort {

    int[] arr;
    int size;
    public heapsort(int[] arr){
        this.arr=arr;
        size= this.arr.length;
    }

    public static void main(String[] args) {
        heapsort heap = new heapsort(new int[]{3, 23, 5, 6, 3, 1, 6, 3});
        int[] sort = heap.sort();
        System.out.println(Arrays.toString(sort));
    }
    public int[] sort(){
        //建堆
        int last=(arr.length-2)>>1;
        while(last>=0){
            siftDown(last);
            last--;
        }
        //此为最小堆
        //移除
        //堆排序要返回新的数组，不能返回原来的。
        int[] res=new int[arr.length];
        for(int i=0;i< arr.length;i++){
            res[i]=removeMin();
        }
        return res;
    }

    public int removeMin(){
        int res=arr[0];
        arr[0]=arr[size-1];
        size--;
        siftDown(0);
        return res;
    }

    public void siftDown(int start){
        while(leftChild(start)<size){
            int j=leftChild(start);
            if(j+1<size&&arr[j]>arr[j+1]) j++;
            if(arr[start]<=arr[j]) break;
            if(arr[j]<arr[start]){
                swap(j,start);
                start=j;
            }
        }
    }

    public void swap(int i,int j){
        int tmp=arr[j];
        arr[j]=arr[i];
        arr[i]=tmp;
    }
    public int leftChild(int i){
        return 2*i+1;
    }
}

原地堆排序

原地堆排序的关键在于siftDown时不是单单把末尾元素复制到arr[0]，还需把arr[0]复制到末尾，即交换首尾元素。

public class heapsort {

    int[] arr;
    int size;
    public heapsort(int[] arr){
        this.arr=arr;
        size= this.arr.length;
    }

    public static void main(String[] args) {
        heapsort heap = new heapsort(new int[]{3, 23, 5, 6, 3, 1, 6, 3});
        heap.sort();
        System.out.println(Arrays.toString(heap.arr));
    }
    public void sort(){
        //建堆
        int last=(arr.length-2)>>1;
        while(last>=0){
            siftDown(last);
            last--;
        }
        //此为最小堆
        //移除
        for(int i=0;i< arr.length;i++){
            removeMin();
        }
    }

    public int removeMin(){
        int res=arr[0];
        //交换末尾和arr[0]
        swap(0,size-1);
        size--;
        siftDown(0);
        return res;
    }

    public void siftDown(int start){
        while(leftChild(start)<size){
            int j=leftChild(start);
            if(j+1<size&&arr[j]>arr[j+1]) j++;
            if(arr[start]<=arr[j]) break;
            if(arr[j]<arr[start]){
                swap(j,start);
                start=j;
            }
        }
    }

    public void swap(int i,int j){
        int tmp=arr[j];
        arr[j]=arr[i];
        arr[i]=tmp;
    }
    public int leftChild(int i){
        return 2*i+1;
    }
}

Topk问题

Top K 问题就是求解前 K 个最大的元素或者最小的元素。元素个数不确定，数据量可能很大，甚至源源不断到来，但需要知道目前为止前 K 个最大或最小的元素。当然问题还可能变为求解第 K 个最大的元素或最小的元素。

通常我们有如下解决方案：

  1. 使用JDK中自带的排序，如Arrays.sort()，由于底层使用的快速排序，所以时间复杂度为O(nlogn)。
  2. 利用堆。

方法一是在元素个数已经确定的情况下可以使用，而堆可支持源源不断的数据从而求得目前为止的top k。

具体利用堆求topk的思路如下：

我们维护一个长度为 K 的数组，最前面 K 个元素就是目前最大的 K 个元素，以后每来一个新元素，都先找数组中的最小值，将新元素与最小值相比，如果小于最小值，则什么都不变，如果大于最小值，则将最小值替换为新元素。这样一来，数组中维护的永远是最大的 K 个元素，不管数据源有多少，需要的内存开销都是固定的，就是长度为 K 的数组。不过，每来一个元素，都需要找到最小值，进行 K 次比较，是否有办法能减少比较次数呢？

当然，这时候堆就要登场了，我们使用最小堆维护这 K 个元素，每次来新的元素，只需要和根节点比较，小于等于根节点，不需要变化，否则用新元素替换根节点，然后siftDown调整堆即可。此时的时间复杂度为O(nlogK)，相比上述两种方法，效率大大提升，且空间复杂度也大大降低。

public int[] topK(int[] arr,int k){
	Queue<Integer> queue=new PriorityQueue<>((a1,a2)->{
        return a1-a2;
    });
    for(int i=0;i<arr.length;i++){
        if(queue.size()==k){
            if(arr[i]>queue.peek()){
                queue.poll();
                queue.offer(arr[i]);
            }
        }else {
            queue.offer(arr[i]);
        }
    }
    int index=0;
    int[] res=new int[k];
 	while(!queue.isEmpty()){
        res[index++]=queue.poll();
    }
    return res;
}

求解中位数

堆也可以用于求解中位数，数据量可能很大且源源不断到来。

注意：如果元素个数是偶数，那么我们假定中位数取任意一个都可以。

求中位数有两种思路

  • 只使用一个堆，类似于求topk的思路

一个数组中的中位数，当数组长度为奇数时，即为第arr.length/2大的数，当为偶数时，中位数即(arr[arr.length/2-1]+arr[arr.length/2])/2,即向数组中再取一个数即可。故我们可以理解为维护一个size/2+1的堆。根据数组长度判断该返回什么中位数。

class Solution {
     Queue<Integer> queue=new PriorityQueue<>((a1,a2)->{
            return a1-a2;
        });
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        int len=nums1.length+nums2.length;
        int k=len/2+1;

        for(int num:nums1){
            add(num,k);
        }
        for(int num:nums2){
            add(num,k);
        }

        return (len & 1)==0?((double)queue.poll()+queue.poll())/2:queue.poll();
    }

    public void add(int num,int k){
        if(queue.size()==k){
            if(queue.peek()<num){
                queue.poll();
                queue.offer(num);
            }
        }else {
            queue.offer(num);
        }
    }
}
  • 第二种方式是维护两个堆，一个最大堆，一个最小堆。
    • 添加的第一个元素作为中位数 m，最大堆维护 <= m 的元素，最小堆维护 >= m 的元素，两个堆都不包含 m。
    • 当添加第二个元素 e 时，将 e 与 m 比较，若 e <= m，则将其加入到最大堆中，否则加入到最小堆中。
    • 如果出现最小堆和最大堆的元素个数相差 >= 2，则将 m 加入元素个数少的堆中，然后让元素个数多的堆将根节点移除并赋值给 m。
    • 以此类推不断更新。
class Solution {
   Queue<Integer> maxQ=new PriorityQueue<>((a1,a2)->{
        return a2-a1;
    });
    Queue<Integer> minQ=new PriorityQueue<>();
    Integer m;
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        for(int num:nums1){
            add(num);
        }
        for(int num:nums2){
            add(num);
        }
        if(minQ.size()<maxQ.size()){
            minQ.add(m);
        }else if(maxQ.size()<minQ.size()){
            maxQ.add(m);
        }
        int len=nums1.length+nums2.length;
        return (len&1)==0?((double)maxQ.peek()+minQ.peek())/2:m;
    }

    public void add(Integer num){
        if(m==null){
            m=num;
            return;
        }
        if(m>num){
            maxQ.add(num);
        }else {
            minQ.add(num);
        }
        if(maxQ.size()-minQ.size()>=2){
            minQ.add(m);
            m=maxQ.poll();
        }else if(minQ.size()-maxQ.size()>=2){
            maxQ.add(m);
            m=minQ.poll();
        }
    }
}
