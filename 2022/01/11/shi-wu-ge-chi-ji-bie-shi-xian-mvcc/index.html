前言

上一章我们讲到一个事务更新一行时如果刚好有另一个事务拥有这一行的行锁，他会被锁住，进入等待。问题是，进入等待之后，等到这个事务获取到自己的行锁时要更新数据他读到的值是什么呢？

mysql> CREATE TABLE `t` (
  `id` int(11) NOT NULL,
  `k` int(11) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB;
insert into t(id, k) values(1,1),(2,2);

注意begin命令并不是事务的起点，而是等到第一个操作InnoDB表语句时事务才真正启动，如果要马上启动一个事务，可以使用start transaction with consistent snapshot

上文中事务B查到的是k=3，事务A查到的k=1，你也许会问为什么？
